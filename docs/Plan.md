## 2. Перечень используемых инструментов с обоснованием выбора.

1. **IntelliJ IDEA Community Edition 2023.2.2** – гибкая и удобная среда разработки с поддержкой Java и всех актуальных
   технологий и фреймворков.
2. **Java 17** – язык программирования (в нашем случае для написания автотестов). Имеет набор готового ПО для
   разработки и запуска приложений.
3. **Postman** – это программа, которая предоставляет вам все необходимые инструменты для тестирования **API**:
    возможность посылать запросы, писать документацию к **API**, запускать автотесты, составлять иерархию тестов,
    проводить версионирование и так далее.
4. **Gradle** – система сборки проекта, с помощью которой возможно управление подключёнными зависимостями и генерация
   отчётов о тестировании.
5. **Docker** – это программное обеспечение для разработки, доставки и запуска контейнерных приложений. Он позволяет
   создавать контейнеры, автоматизировать их запуск и развертывание, управляет жизненным циклом. С помощью **Docker**
   можно запускать множество контейнеров на одной хост-машине.
6. **Selenide** – фреймворк для автоматизированного тестирования веб-страниц и веб-приложений, который позволяет быстро
   и просто его использовать при написании тестов.
7. **Git** и **GitHub** – для ведения репозитория по проектам. **Git** очень распространенная система контроля версий,
    поэтому достаточно хорошо взаимодействует с различными ОС. **GitHub** специализированный веб-сервис с удобным
    интерфейсом, интегрирован с **Git**.
8. **AppVeyor Continuous Integration** – веб-сервис, предназначенный для сборки и тестирования программного обеспечения
    расположенного на **GitHub**, использующий виртуальные машины **Microsoft** **Windows** и **Ubuntu**. Бесплатный и
    удобный сервис, который может осуществлять сборку как под управлением **Linux**, так и под **Windows**, а если
    необходимо то и под несколькими сразу.
9. **DataGrip 2023.2.1** – это мощный инструмент для работы с базами данных. Он поддерживает самые разные базы данных, включая **PostgreSQL**, **MySQL**, **SQL Server**, **Oracle**, **MongoDB**, а также источники данных DDL.

## 3. Перечень и описание возможных рисков при автоматизации.

  также настроить работу симулятора банковских сервисов.
- Отсутствие постановки задач к тестируемому функционалу приложения.
- Отсутствие документации по API.
- Неуникальные локаторы, которые затрудняют поиск элементов.
- Утрата актуальности автотестов при изменениях структуры приложения.
